## Was sie machen
Mainline is a Git-native memory layer for coding agents. It gives agents and reviewers repo memory before the diff: prior decisions, constraints, abandoned approaches, validation notes, and related in-flight work. AI agents make code cheap to produce and harder to review — Mainline makes the intent reviewable before the generated code lands. An intent captures the user goal, why the work exists, decisions and rejected alternatives, validation and review notes, explicit constraints, risks and follow-ups, related files and subsystems, in-flight overlap with other agents or teammates, and the commit that eventually carried the work onto main. Mainline is repo-local engineering memory that travels with code through Git refs and Git notes, readable via CLI commands and a hosted Hub.
